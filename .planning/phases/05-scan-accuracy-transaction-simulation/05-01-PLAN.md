# Plan 05-01: Fix ALT Resolution in scan.ts

## Objective

`extractAccounts()` in scan.ts only reads `tx.message.staticAccountKeys`, missing all accounts loaded via Address Lookup Tables. Modern Solana drainers use ALTs specifically to hide malicious program/destination addresses from naive scanners. This fix resolves all ALTs before threat scoring so those hidden accounts are included.

## Context

The same `resolveALTs` pattern already exists in `packages/sdk/src/tx-decoder.ts` (lines 99ŌĆō121). Replicate it in scan.ts ŌĆö do NOT move it to a shared package (avoids cross-package dep change).

The fix requires `connection.getAddressLookupTable(lookup.accountKey)` for each entry in `tx.message.addressTableLookups`. The connection must be async, so `extractAccounts()` becomes async.

## Tasks

<task id="1" name="Add async ALT resolution to extractAccounts">

Replace the synchronous `extractAccounts(tx)` in `apps/worker/src/scan.ts` with an async version that:

1. Adds a `Connection` import from `@solana/web3.js`
2. Adds a `getConnection()` helper (same pattern as tx-decoder.ts: `new Connection('https://mainnet.helius-rpc.com/?api-key=' + process.env.HELIUS_API_KEY, 'confirmed')`)
3. Makes `extractAccounts` async, accepting `(tx: VersionedTransaction, connection: Connection): Promise<PublicKey[]>`
4. After collecting staticAccountKeys, loops `tx.message.addressTableLookups` and resolves each ALT:
   ```ts
   for (const lookup of tx.message.addressTableLookups) {
     try {
       const alt = await connection.getAddressLookupTable(lookup.accountKey)
       if (!alt.value) continue
       for (const idx of lookup.writableIndexes) accounts.push(alt.value.state.addresses[idx])
       for (const idx of lookup.readonlyIndexes) accounts.push(alt.value.state.addresses[idx])
     } catch { /* non-fatal ŌĆö skip this ALT */ }
   }
   ```
5. Updates the two call sites in the `handler` function to `await extractAccounts(tx, connection)` (create connection once at top of handler, reuse for both calls)
6. Deduplication logic stays the same

Commit: `fix(scan): resolve Address Lookup Tables before threat scoring`

</task>
