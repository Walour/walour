# Plan 05-02: Token-2022 PermanentDelegate Detection

## Objective

Add detection for the Token-2022 `PermanentDelegate` extension. When a mint has PermanentDelegate set, the mint authority can drain any holder's token account at any time — most users are unaware this exists. Detecting it at the instruction level and surfacing it as a red flag gives users critical information before signing.

## Context

- Token-2022 program: `TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb` (already in `TOKEN_2022_PROGRAM` const)
- PermanentDelegate instruction discriminator: `'1c'` (hex for 28, the `InitializePermanentDelegate` instruction index in Token-2022)
- Detection pattern: same as existing SetAuthority (`'06'`) and CloseAccount (`'09'`) checks
- File: `packages/sdk/src/tx-decoder.ts`, `detectRedFlags()` function

## Tasks

<task id="1" name="Add PermanentDelegate red flag detection">

   ```ts
   interface RedFlag {
   ```

   ```ts
       type: 'permanent_delegate',
       detail: 'Token mint has PermanentDelegate — issuer can drain any holder account at any time',
     })
   }
   ```
